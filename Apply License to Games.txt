These are all applied to the HTML games.
TEST BEFORE DEPLOYING TO WEBSITE.

1) At the end of the <style> section apply this (including comments):

  /* License MIT used in this file */
  #license {
      position: fixed;
      bottom: 5px;
      right: 10px;
      font-size: 10px;
      opacity: 0.5;
  }
  #license a {
      color: inherit;
      text-decoration: none;
  }
  #license a:hover {
      text-decoration: underline;
  }  
  /* License */


2)Toward the very end of the file, just below the closing </body> tag, add this:

<div id="license">
    <a href="../../LICENSE.txt">MIT License</a>
</div>


3) Run it locally. You should see the license. THE LINK WILL NOT WORK. That is OK.

Note: I have seen one case where this did not work. It failed because there was no
default color defined in the app's style. It went to the default dark blue for a 
link, with a black background. It could not be seen. To fix, I used this as a
replacement for Step 1:
  /* License MIT used in this file */
  #license {
      position: fixed;
      bottom: 8px;
      right: 12px;
      font-size: 11px;
      opacity: 0.5;
  }
  #license a {
      color: #aaaaaa;
      text-decoration: none;
  }
  #license a:hover {
      text-decoration: underline;
  }  
  /* License */
One might want to change that color, depending on the background.


4) Copy the file to the jjermanis.github.io site where the games get deployed.
\jjermanis.github.io\projects\PROJECT_FOLDER
This should also be tested, and THE LINK SHOULD WORK.
